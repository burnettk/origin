Creating a patch:

A Solaris patch is a package with a few extras.  The first thing you need to get is a unique patch ID.  You can get a patch ID allocated to yourself by running
/net/pip-tools.eng/latest/getpatchid
in a terminal window.  It asks you a few simple questions, and then it generates your patch id.

The following is the structure of a patch.  The files that are italicized are ones that are interesting to us:

Descriptions of files:

Install.info

This is a standard text file for a patch.  It's not needed for the functioning of a patch, but it is like a README file with instructions to add a patch to a system.
 

backoutpatch

This is a script that is already written, and we do not need or want to modify it.
 

installpatch


Again, this is a script that is already written, and we do not need or want to modify it.
 

postbackout


This is a script that is patch-specific, and can be considered analagous to a postremove script for a package.  In this particular patch, the script is just

#!/bin/sh
exit 0
 

postpatch

This is a script that is patch-specific, and can be considered analagous to a postinstall script.  This script needs to be interactive, containing the functionality that a package would put into the request and postinstall scripts.  This script contains:
 

#!/bin/sh

#========================================================
# SETUP
 

# Are we in a chroot?
CHROOT_BASE=/
if [ -f /var/sadm/pkg/SUNWsmscr/pkginfo ]
 then
  CHROOT_BASE=`grep 'CHROOT_TARGET_DIR=' /var/sadm/pkg/SUNWsmscr/pkginfo | awk -F= '{print $2}'`
  echo Chroot detected at $CHROOT_BASE
 fi

PKGINFO=$CHROOT_BASE/var/sadm/pkg/SUNWsmssu/pkginfo

IDENTITY_HOME=`grep '^IDENTITY_HOME=' $PKGINFO | sed 's/IDENTITY_HOME=//'`
BASEDIR=`grep '^BASEDIR=' $PKGINFO | sed 's/BASEDIR=//'`
DIRECTORY_HOST=`grep '^DIRECTORY_HOST=' $PKGINFO | sed 's/DIRECTORY_HOST=//'`
DIRECTORY_PORT=`grep '^DIRECTORY_PORT=' $PKGINFO | sed 's/DIRECTORY_PORT=//'`
AMADMIN_DN=`grep '^AMADMIN_DN=' $PKGINFO | sed 's/AMADMIN_DN=//'`
AMADMIN_PW=`grep '^AMADMIN_PW=' $PKGINFO | sed 's/AMADMIN_PW=//'`
DMANAGER_PW=`grep '^DMANAGER_PW=' $PKGINFO | sed 's/DMANAGER_PW=//'`

DMANAGER_DN='cn=Directory Manager'
 

BASEDIR=$CHROOT_BASE/$BASEDIR
IDENTITY_HOME=$CHROOT_BASE/$IDENTITY_HOME
 

yesno() {
  ok="no"
  question=$1
  default=`echo $2 | cut -c1 | sed y/YN/yn/`
  while [ "$ok" = "no" ]; do
    if [ "$default" = "y" ]; then
    echo "$question [y]/n \c"
    default="y"
    elif [ "$default" = "n" ]; then
    echo "$question y/[n] \c"
    default="n"
    else
      echo "$question y/n \c"
    default=""
    fi

    read ans

    if [ "$ans" = "" ]; then
    if [ "$default" != "" ]; then
    ans=$default
    ok="yes"
    fi
    else
      ans=`echo $ans | cut -c1 | sed y/YN/yn/`
    if [ "$ans" = "y" ] || [ "$ans" = "n" ]; then
    ok="yes"
    fi
    fi
    if [ "$ok" = "no" ]; then
    echo ""
      echo "Invalid response"
    echo ""
    fi
  done
}
show_vars(){
echo "BASEDIR=$BASEDIR"
echo "IDENTITY_HOME=$IDENTITY_HOME"
echo "DIRECTORY_HOST=$DIRECTORY_HOST"
echo "DIRECTORY_PORT=$DIRECTORY_PORT"
echo "AMADMIN_DN=$AMADMIN_DN"
echo "AMADMIN_PW=$AMADMIN_PW"
echo "DMANAGER_DN=$DMANAGER_DN"
echo "DMANAGER_PW=$DMANAGER_PW"
}
get_ans(){
  message=$1
  ans=$2
  echo "$message [$ans]: \c"
  read TEST
  if [ "$TEST" != "" ]; then
    ans=$TEST
  fi
}
 
 

while [ 1 ]
do
    show_vars
  yesno "Use this configuration" "y"
  if [ "$ans" = "n" ]; then
    get_ans "Identity Server Home (example: /opt/SUNWam) " "$IDENTITY_HOME"
    IDENTITY_HOME=$ans
    get_ans "BASEDIR " "$BASEDIR"
    BASEDIR=$ans
    get_ans "Directory Server Host " "$DIRECTORY_HOST"
    DIRECTORY_HOST=$ans
    get_ans "Directory Server Port " "$DIRECTORY_PORT"
    DIRECTORY_PORT=$ans
    get_ans "amAdmin DN " "$AMADMIN_DN"
    AMADMIN_DN=$ans
    get_ans "amAdmin password " "$AMADMIN_PW"
    AMADMIN_PW=$ans
    get_ans "Directory Manager DN " "$DMANAGER_DN"
    DMANAGER_DN=$ans
    get_ans "Directory Manager password " "$DMANAGER_PW"
    DMANAGER_PW=$ans

    else
    break
  fi
done
 
 

LD_LIBRARY_PATH=$IDENTITY_HOME/lib/solaris/sparc
export LD_LIBRARY_PATH

# Initialize the installation log.

INSTALL_LOG=$BASEDIR/logs/sams-users-install.log
/bin/rm -f $INSTALL_LOG
cat /dev/null > $INSTALL_LOG

logmsg() {
   echo [`date +"%Y/%M/%d %H:%M:%S %Z"`]" $1" >> $INSTALL_LOG
   echo "$1"
}

# Edit scripts and add the directory server info.

replace_param() {
   sed "s+$1+$2+g" $3 > $3-new
   mv $3-new $3
}

insert_directory_info() {
   replace_param DIRECTORY_HOST "$DIRECTORY_HOST" $1
   replace_param DIRECTORY_PORT "$DIRECTORY_PORT" $1
   replace_param IDENTITY_HOME "$IDENTITY_HOME" $1
   replace_param AMADMIN_DN "$AMADMIN_DN" $1
   chmod 555 $1
}

insert_directory_info $BASEDIR/bin/sams-user-checks.sh

# Define the file system directories where release1 and release2 users info can be found.

RELEASE1=$BASEDIR/users/release1
RELEASE2=$BASEDIR/users/release2

# Define functions to use when searching and modifying the directory server.

check_directory () {
   $BASEDIR/bin/sams-user-checks.sh $AMADMIN_PW >> $INSTALL_LOG 2>&1
}

modify_directory () {
   if [ ! -f $1 ]; then
      logmsg "ERROR: $1 does not exist."
   else
      echo `date`" Modifying directory with $1" >> $INSTALL_LOG 2>&1
      $IDENTITY_HOME/bin/ldapmodify -h $DIRECTORY_HOST -p $DIRECTORY_PORT -D "$DMANAGER_DN" -w $DMANAGER_PW -a -c -f $1 >> $INSTALL_LOG 2>&1
   fi
}

#========================================================

logmsg "Starting Users install into ldap://"$DIRECTORY_HOST":"$DIRECTORY_PORT

# USER DELETES

logmsg "Removing all old SAMS admin users."

modify_directory $RELEASE1/sams-all-users-orig-delete.ldif

logmsg "Ensuring all AppSvc users are not in directory."

modify_directory $RELEASE1/sams-appsvcs-users-delete.ldif

check_directory
 

# USER ADDS (New with Delete then Add)

logmsg "Adding SAMS admin users."

modify_directory $RELEASE1/sams-admin-users.ldif

#logmsg "Adding all users under o=App_Services."

modify_directory $RELEASE1/sams-appsvcs-users.ldif
 
 

# ROLE ASSIGNS (New)

logmsg "Assigning SAMS admin roles to SAMS admin users."

modify_directory $RELEASE1/sams-admin-roles-assign.ldif
#modify_directory $RELEASE2/sams-admin-roles-assign.ldif

check_directory
 

logmsg "Users install finished."

echo "Users installation log is at "$INSTALL_LOG

.diPatch


The existence of this file tells patchadd that this patch is a "direct injection" patch.  I'm not sure what that means, but I couldn't get a patch to work without doing this.
 

pkginfo


The pkginfo is a standard pkginfo file, with a few special quirks:

PKG is the name of the package that this patch will update.
VERSION is the version of that package that this patch will update.  A patch will only install if that version of the package is installed.
SUNW_PATCHID is the only new field n this pkginfo file.
 

PKG=SUNWsmssu
NAME=Sun Microsystems SAMS Users
ARCH=SPARC
PRODNAME=Sun Microsystems SAMS Users package
DESC=This package contains everything needed to install Users specific to SAMS.
VENDOR=Sun Microsystems Inc.
CLASSES=none
CATEGORY=Application
VERSION=015
BASEDIR=/opt/SUNWsms
IDENTITY_HOME=/opt/SUNWam
DIRECTORY_HOST=gsoit-sandbox1.central.sun.com
DIRECTORY_PORT=10389
DMANAGER_DN="cn=Directory Manager"
DMANAGER_PW=password
AMADMIN_DN="uid=amadmin,ou=People,o=Sun,dc=sun,dc=com"
AMADMIN_PW=password
SUNW_PATCHID=114900-01
